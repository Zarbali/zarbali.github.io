# Как полностью самостоятельно сделать страницу проекта

Этот документ — пошаговая инструкция на русском языке. По ней вы сможете в следующий раз сами сделать такую же страницу проекта (отдельная страница с описанием и демо) без помощи. Объяснено: что за что отвечает, какие данные куда писать, как устроен код и что проверять.

---

## Часть 1. Что мы делаем и зачем

### Идея

- На главной странице есть **карточки проектов**.
- По клику на проект открывается **отдельная страница** с полным описанием.
- На странице проекта могут быть: заголовок, текст, теги технологий, кнопка «На GitHub», блок **«Попробовать»** (форма + результат), команда запуска и пример вывода.

### Почему одна страница для всех проектов

- Один файл `project.html` — шаблон для всех проектов.
- В адресе передаётся **параметр**: `project.html?project=название_проекта` (название = **slug**).
- Контент каждого проекта хранится в данных (в объекте по slug). Так не нужно создавать отдельный HTML для каждого проекта.

### Какие файлы участвуют

| Файл | Назначение |
|------|------------|
| **index.html** | Главная страница; на ней блок с карточками проектов. |
| **project.html** | Одна страница проекта для всех проектов. |
| **js/data.js** | Массив **projectsData** — список проектов для карточек (короткое описание, ссылки, slug). |
| **js/projectDetails.js** | Объект **projectDetails** — полное описание каждого проекта по slug (текст, демо, команда запуска). |
| **js/render.js** | Рисует карточки на главной и формирует ссылки «View Project» и «GitHub». |
| **css/components.css** | Стили карточек и **все стили страницы проекта** (классы `project-detail__...` и `project-detail__demo...`). |

Дальше разберём каждый кусок по отдельности, чтобы вы могли повторить или изменить его сами.

## Часть 2. Данные для карточек на главной (`js/data.js`)

Карточки на главной строятся из массива **`projectsData`**. Один элемент массива = один проект.

### Обязательные поля

- **slug** — уникальный идентификатор проекта, только латиница, без пробелов (например: `healthylifestylega`, `my-second-project`).  
  Ссылка на страницу проекта будет: `project.html?project=ВАШ_SLUG`.
- **title** — название на карточке.
- **description** — короткое описание (1–2 предложения).
- **tags** — массив строк (технологии/темы), например: `['Python', 'AI']`.
- **links** — объект с полями:
  - **page** — ссылка на страницу проекта. Формат: `'project.html?project=' + slug`. Если страницы ещё нет — `'#'`.
  - **code** — ссылка на репозиторий (GitHub и т.п.). Если нет — `'#'`.
- **image** — эмодзи или текст для картинки карточки (например: `'🥗'`, `'🚀'`).

### Пример одной записи

```javascript
{
  slug: 'healthylifestylega',
  title: 'Healthy Lifestyle GA',
  description: 'AI-based recommendation system using a Genetic Algorithm to generate personalized diet and workout plans.',
  tags: ['Python', 'Genetic Algorithm', 'AI', 'pandas'],
  links: {
    page: 'project.html?project=healthylifestylega',
    code: 'https://github.com/Zarbali/HealthyLifestyleGA'
  },
  image: '🥗'
}
```

Важно: **slug в `projectsData`** и **ключ в `projectDetails`** должны совпадать. Иначе по ссылке «View Project» откроется страница, но контент не найдётся.

## Часть 3. Данные для страницы проекта (`js/projectDetails.js`)

Здесь хранится полное описание проекта. Ключ объекта — **тот же slug**, что и в `projectsData`.

### Структура одного проекта в `projectDetails`

```javascript
projectDetails = {
  ВАШ_SLUG: {
    title: 'Название на странице проекта',
    subtitle: 'Короткий подзаголовок (один абзац).',
    techStack: ['Python', 'pandas', 'AI'],
    githubUrl: 'https://github.com/...',
    howToRun: 'python main.py',
    exampleOutput: 'Текст примера вывода программы...',
    hasDemo: true,
    demoConfig: { ... },
    sections: [ ... ]
  }
};
```

### Поля по назначению

| Поле | Обязательно | Описание |
|------|-------------|----------|
| **title** | Да | Заголовок страницы проекта. |
| **subtitle** | Нет | Подзаголовок (один абзац под заголовком). |
| **techStack** | Нет | Массив строк — теги технологий под подзаголовком. |
| **githubUrl** | Да (если есть кнопка GitHub) | Ссылка на репозиторий. |
| **howToRun** | Нет | Команда запуска (одна строка). |
| **exampleOutput** | Нет | Пример вывода программы (многострочная строка). |
| **hasDemo** | Нет | `true` — показывать блок «Попробовать» с формой. |
| **demoConfig** | Если hasDemo: true | Объект с настройками формы и готовыми ответами. |
| **sections** | Нет (но обычно есть) | Массив секций с текстом описания. |

### Секции (`sections`)

Каждая секция — объект:

```javascript
{
  title: 'Заголовок секции',
  content: 'Основной текст секции. Может быть длинным.',
  bullets: [
    'Пункт списка 1',
    'Пункт списка 2'
  ]
}
```

- **title** — заголовок секции.
- **content** — текст (один или несколько абзацев).
- **bullets** — необязательно. Если есть — под текстом выводится маркированный список.

### Демо (`demoConfig`)

Демо не запускает ваш код на сервере. Оно просто показывает форму и по выбору пользователя подставляет **заранее заготовленный** ответ. Так можно «протестировать идею» прямо на сайте.

Структура:

```javascript
demoConfig: {
  goals: [
    { value: 'weight_loss', label: 'Weight loss' },
    { value: 'muscle_gain', label: 'Muscle gain' }
  ],
  preferences: [
    { value: 'vegan', label: 'Vegan' },
    { value: 'vegetarian', label: 'Vegetarian' }
  ],
  results: {
    'weight_loss_vegan': {
      diet: 'Low-Cal Vegan (1800 kcal)',
      workout: 'Cardio (350 kcal)',
      net: '1450'
    },
    'weight_loss_vegetarian': { ... },
    'muscle_gain_vegan': { ... }
    // ключ = цель_предпочтение (как в value у goals и preferences)
  }
}
```

- **goals** — варианты для первого выпадающего списка (value — значение для кода, label — подпись на экране).
- **preferences** — варианты для второго выпадающего списка.
- **results** — объект: ключ = `цель_предпочтение` (например `weight_loss_vegan`), значение = объект с полями `diet`, `workout`, `net` (или любыми другими — главное, как вы их потом выводите в скрипте).

Для каждого сочетания «цель + предпочтение» должна быть своя запись в `results`, иначе при выборе этого сочетания можно показать, например, «Нет рекомендации».

## Часть 4. Как карточки превращаются в ссылки (`js/render.js`)

В `render.js` метод **`renderProjects(projects)`** получает массив `projectsData` и для каждого проекта создаёт HTML карточки.

Важно для страницы проекта:

- Ссылка **«View Project»** должна вести на страницу проекта.  
  Адрес берётся из `project.links.page`. Если там `'project.html?project=healthylifestylega'`, пользователь перейдёт на эту страницу с параметром `project=healthylifestylega`.
- Ссылка **«GitHub»** берётся из `project.links.code` и открывается в новой вкладке (`target="_blank"`).

Если у вас уже есть такой рендер и вы добавляете в объект проекта поле **links.page** (и при необходимости **links.code**), карточка автоматически получит нужные ссылки. Проверьте только, что в разметке карточки используются именно `project.links.page` и `project.links.code` (или как у вас названы поля).

Текст ссылок может быть «View Project» и «GitHub» — или свой, главное чтобы `href` был правильный.

## Часть 5. Страница проекта `project.html`: что в ней должно быть

### 5.1 Разметка (HTML)

- Обычная шапка сайта (навигация), как на главной, но ссылки ведут на `index.html#about`, `index.html#projects` и т.д.
- Ссылка «Назад к проектам»: `index.html#projects`.
- Пустой контейнер, куда будет вставляться контент, например:  
  `<div id="projectContent"></div>`
- Блок «Проект не найден» (изначально скрыт), например:  
  `<div id="projectNotFound" style="display: none;">Проект не найден. <a href="index.html#projects">К списку проектов</a></div>`
- Подвал (footer).

Подключить стили те же, что на главной (чтобы страница проекта выглядела в одном стиле), и скрипты: `projectDetails.js` и скрипт, который будет читать параметр из URL и рисовать контент (см. ниже).

### 5.2 Логика в скрипте (по шагам)

Скрипт на странице проекта должен делать следующее.

**Шаг 1. Получить slug из адреса**

```javascript
var params = new URLSearchParams(window.location.search);
var slug = params.get('project');
```

- `window.location.search` — это строка вида `?project=healthylifestylega`.
- `params.get('project')` — значение параметра `project`, т.е. slug.

**Шаг 2. Проверить, есть ли такой проект**

```javascript
if (!slug || !window.projectDetails || !projectDetails[slug]) {
  // спрятать контейнер контента, показать «Проект не найден»
  return;
}
var project = projectDetails[slug];
```

**Шаг 3. Собрать HTML из данных проекта**

- Заголовок: `project.title`.
- Подзаголовок: `project.subtitle` (если есть).
- Теги технологий: цикл по `project.techStack`, каждый элемент — например `<span class="project-detail__tag">...</span>`.
- Кнопки: «View on GitHub» (href = `project.githubUrl`), «Back to Projects» (href = `index.html#projects`).
- Если `project.hasDemo` и есть `project.demoConfig` — вставить блок демо:
  - заголовок и описание демо;
  - форма: два `<select>` (цель и предпочтение) из `demoConfig.goals` и `demoConfig.preferences`;
  - кнопка «Get recommendation» (или свой текст);
  - пустой блок с id для результата (например `id="demoResult"`).
- Секции: цикл по `project.sections`. Для каждой секции — заголовок (`section.title`), абзац (`section.content`), при наличии — список (`section.bullets`).
- Если есть `project.howToRun` — секция «How to run» и блок с текстом команды.
- Если есть `project.exampleOutput` — секция «Example output» и блок с примером вывода.

**Шаг 4. Вставить собранный HTML**

```javascript
document.getElementById('projectContent').innerHTML = html;
```

И обновить заголовок вкладки: `document.title = project.title + ' | ...';`

**Шаг 5. Защита от XSS**

Перед вставкой любой текст из данных нужно экранировать (заменить `<`, `>`, `"`, `'` и т.д.), иначе если в описании проекта будет, например, `<script>`, он выполнится. Функция может быть такой:

```javascript
function escapeHtml(text) {
  var div = document.createElement('div');
  div.textContent = text;
  return div.innerHTML;
}
```

Использовать её для всех полей, которые приходят из данных (title, subtitle, content, варианты в форме и т.д.), кроме тех случаев, где вы сознательно вставляете безопасный HTML.

**Шаг 6. Подключить демо (если есть)**

После вставки HTML найти форму и блок результата по id. На форму повесить обработчик `submit`:

- Вызвать `event.preventDefault()`.
- Прочитать выбранные значения из двух `<select>` (цель и предпочтение).
- Собрать ключ: `var key = goal + '_' + preference;`
- Взять результат: `var r = project.demoConfig.results[key];`
- Показать в блоке результата «Calculating...» (или другой текст), через небольшую задержку (например 500–600 мс) заменить на текст из `r` (diet, workout, net или как у вас названы поля). Если `r` нет — показать «Нет рекомендации» или аналог.

Так вы «полностью самостоятельно» реализуете ту же логику, что и в текущем проекте.

## Часть 6. Стили страницы проекта (CSS)

Все классы для страницы проекта лучше держать в одном месте (например в `components.css`), с общим префиксом, например **`project-detail__`** и **`project-detail__demo`**.

Минимально нужны:

- Контейнер страницы: отступы сверху/снизу, учёт фиксированной шапки.
- Заголовок (`.project-detail__title`), подзаголовок (`.project-detail__lead`).
- Блок тегов (flex/grid), стиль одного тега (`.project-detail__tag`).
- Кнопки (можно переиспользовать классы кнопок с главной).
- Секции: отступ между секциями, заголовок секции, абзац, список (`.project-detail__bullets`).
- Блок кода (команда запуска и пример вывода): моноширинный шрифт, фон, рамка (`.project-detail__code-block`).
- Блок демо: контейнер (`.project-detail__demo`), заголовок и описание, форма (строка с подписью и select), кнопка, блок результата (`.project-detail__demo-result`), при необходимости класс для видимости результата (например `.project-detail__demo-result--visible`) и стиль для «Calculating...».

Если опишете структуру HTML (какие классы вы используете), можно перенести эти имена в документ как эталон — тогда «сделать так же» будет сводиться к копированию структуры и классов.

## Часть 7. Пошаговый план: «делаю с нуля»

Следуя этому плану, вы сможете сделать страницу проекта полностью самостоятельно.

1. **Данные для карточек**
   - В `data.js` в массив `projectsData` добавить объект проекта с полями: `slug`, `title`, `description`, `tags`, `links: { page: 'project.html?project=' + slug, code: '...' }`, `image`.
   - Убедиться, что slug написан латиницей, без пробелов.

2. **Данные для страницы проекта**
   - В `projectDetails.js` в объект `projectDetails` добавить запись с ключом = тем же slug.
   - Заполнить: `title`, `subtitle`, `techStack`, `githubUrl`, `howToRun`, `exampleOutput`, `sections` (массив с `title`, `content`, при необходимости `bullets`).
   - Если нужен блок «Попробовать»: поставить `hasDemo: true`, заполнить `demoConfig` (goals, preferences, results для всех сочетаний).

3. **Ссылки с главной**
   - Убедиться, что в `render.js` у карточки есть ссылка с `href={project.links.page}` (например «View Project») и при необходимости ссылка на код (GitHub) из `project.links.code`.

4. **Страница project.html**
   - Создать файл `project.html` с шапкой, «Назад к проектам», контейнером `#projectContent`, блоком «Проект не найден», подвалом.
   - Подключить `projectDetails.js` и один скрипт, который: читает `?project=`, берёт проект из `projectDetails[slug]`, собирает HTML (заголовок, подзаголовок, теги, кнопки, демо-форма, секции, How to run, Example output), экранирует текст через `escapeHtml`, вставляет в `#projectContent`, вешает обработчик на форму демо (если есть).

5. **Стили**
   - В CSS добавить классы для всех блоков страницы проекта (см. Часть 6).

6. **Проверка**
   - Открыть главную, нажать «View Project» — должна открыться `project.html?project=ваш_slug` с нужным текстом.
   - Проверить кнопку GitHub.
   - Если есть демо — выбрать цель и предпочтение, нажать «Get recommendation» — должен появиться соответствующий результат из `results`.

## Часть 8. Частые ошибки и что проверить

- **Slug не совпадает:** в `projectsData` slug один, в `projectDetails` ключ другой — страница откроется, но контент не найдётся. Сделайте полное совпадение (включая регистр).
- **Ссылка на страницу проекта:** в `links.page` должно быть ровно `'project.html?project=' + ваш_slug` (или уже подставленный slug). Без лишних пробелов и других параметров.
- **Демо не срабатывает:** проверьте, что ключи в `results` имеют вид `цель_предпочтение` (как в `value` у goals и preferences), и что после вставки HTML форма и блок результата находятся по id (форма не должна быть вложена в другой элемент с тем же id).
- **XSS:** не вставляйте в `innerHTML` сырой текст из данных без `escapeHtml` (или аналога).

## Часть 9. Краткий чек-лист

Перед тем как считать страницу проекта готовой, проверьте:

- [ ] В `data.js` у проекта есть `slug` и `links.page` с путём `project.html?project=SLUG`.
- [ ] В `projectDetails.js` есть запись с ключом = тем же `slug`.
- [ ] На главной по клику «View Project» открывается `project.html` с параметром `project=...`.
- [ ] На странице проекта отображаются заголовок, подзаголовок, теги, кнопки, секции текста.
- [ ] Кнопка «View on GitHub» ведёт на нужный репозиторий.
- [ ] Если есть демо — форма есть, по кнопке показывается результат для выбранной пары цель/предпочтение.
- [ ] Все тексты из данных экранируются перед вставкой в HTML.
- [ ] Стили страницы проекта подключены и выглядят так, как задумано.

Если все пункты выполнены, вы сделали страницу проекта полностью самостоятельно по этой инструкции. В следующий раз можно использовать этот документ как единственный гайд: что в каких файлах лежит, как устроены данные и скрипт, и в каком порядке что делать.
