# Пошаговое описание: как сделана страница проекта

Документ описывает, как по шагам была реализована страница проекта: от данных до отображения и интерактивного демо. Всё на русском языке.

---

## Шаг 1. Идея и общая схема

**Цель:**  
При клике на проект на главной странице открывать отдельную страницу с полным описанием и возможностью «протестировать» идею проекта прямо в браузере (без запуска Python).

**Схема работы:**
1. На главной (`index.html`) показываются карточки проектов из списка `projectsData`.
2. У каждой карточки есть ссылка «View Project» — она ведёт на `project.html?project=название_проекта` (название = slug).
3. Страница `project.html` одна для всех проектов. Она смотрит в адресе параметр `project`, по нему находит описание в объекте `projectDetails` и рисует контент.
4. Если у проекта включено демо (`hasDemo: true`), на странице показывается форма «Попробовать» и блок с результатом.

**Почему одна страница `project.html`?**  
Чтобы не создавать отдельный HTML-файл для каждого проекта. Добавление нового проекта = новая запись в данных, без новых страниц.

## Шаг 2. Данные для карточек на главной (`js/data.js`)

На главной странице карточки берутся из массива **`projectsData`**. Каждый элемент — один проект.

**Что нужно для карточки:**
- **slug** — уникальный идентификатор (латиница, без пробелов). Например: `healthylifestylega`. Он используется в ссылке: `project.html?project=healthylifestylega`.
- **title** — короткое название на карточке.
- **description** — одно-два предложения для карточки.
- **tags** — массив тегов (технологии, темы): `['Python', 'Genetic Algorithm', 'AI', 'pandas']`.
- **links** — объект с двумя ссылками:
  - **page** — ссылка на страницу проекта. Для первого проекта: `'project.html?project=healthylifestylega'`. Если страницы ещё нет — `'#'`.
  - **code** — ссылка на репозиторий (GitHub). Для первого проекта: `'https://github.com/Zarbali/HealthyLifestyleGA'`. Если нет — `'#'`.
- **image** — эмодзи или иконка для карточки (например, 🥗).

**Пример записи в `data.js`:**

```javascript
{
  slug: 'healthylifestylega',
  title: 'Healthy Lifestyle GA',
  description: 'AI-based recommendation system using a Genetic Algorithm...',
  tags: ['Python', 'Genetic Algorithm', 'AI', 'pandas'],
  links: {
    page: 'project.html?project=healthylifestylega',
    code: 'https://github.com/Zarbali/HealthyLifestyleGA'
  },
  image: '🥗'
}
```

Так мы «привязываем» карточку к странице проекта и к GitHub по одному slug.

## Шаг 3. Данные для страницы проекта (`js/projectDetails.js`)

Полное описание проекта и настройки демо хранятся в объекте **`projectDetails`**. Ключ объекта — тот же **slug**, что и в `projectsData` (например, `healthylifestylega`).

**Структура одного проекта в `projectDetails`:**

| Поле | Назначение |
|------|------------|
| **title** | Полное название на странице проекта. |
| **subtitle** | Короткий подзаголовок (один абзац под заголовком). |
| **techStack** | Массив строк — технологии в виде тегов: `['Python', 'pandas', 'Genetic Algorithm', 'CSV data']`. |
| **githubUrl** | Ссылка на репозиторий. |
| **howToRun** | Команда запуска, например: `'python main.py'`. |
| **exampleOutput** | Пример вывода программы (текст в несколько строк). |
| **hasDemo** | `true`, если у проекта есть блок «Попробовать» с формой. |
| **demoConfig** | Настройки демо (см. ниже). |
| **sections** | Массив секций с текстом описания (см. ниже). |

**Секции (`sections`):**  
Каждая секция — объект с полями:
- **title** — заголовок секции (например, «Overview», «Why Python»).
- **content** — основной текст секции (один или несколько абзацев).
- **bullets** — необязательный массив строк. Если он есть, под текстом выводится маркированный список.

**Конфиг демо (`demoConfig`):**  
Нужен только если `hasDemo: true`.

- **goals** — массив вариантов «Цель»:  
  `[ { value: 'weight_loss', label: 'Weight loss' }, ... ]`
- **preferences** — массив вариантов «Предпочтение»:  
  `[ { value: 'vegan', label: 'Vegan' }, ... ]`
- **results** — объект с готовыми ответами. Ключ — пара «цель_предпочтение» (например, `weight_loss_vegetarian`), значение — объект с полями:
  - **diet** — строка с названием диеты и калориями.
  - **workout** — строка с названием тренировки и калориями.
  - **net** — итоговые калории (строка).

Демо не запускает настоящий Python: оно просто по выбранным «Цель» и «Предпочтение» подставляет готовый текст из `results`. Так создаётся эффект «теста» прямо на странице.

## Шаг 4. Как карточки рисуются и куда ведут ссылки (`js/render.js`)

В **`render.js`** метод **`renderProjects`** по массиву `projectsData` строит HTML для каждой карточки.

**Логика ссылок:**
- Ссылка **«View Project»** показывается только если у проекта есть `links.page` и он не равен `'#'`. Атрибут `href` берётся из `project.links.page` (например, `project.html?project=healthylifestylega`). При клике пользователь переходит на страницу проекта.
- Ссылка **«GitHub»** показывается только если `links.code` не равен `'#'`. Она открывается в новой вкладке (`target="_blank"`).

Таким образом, с главной страницы пользователь попадает на страницу проекта по ссылке с параметром `?project=slug`.

## Шаг 5. Страница проекта `project.html`: что в ней есть

В **`project.html`**:
- Одна и та же разметка для всех проектов: навигация, блок «Назад к проектам», контейнер `#projectContent` (сюда вставляется контент), блок «Проект не найден» (`#projectNotFound`), футер.
- Подключаются скрипты: `projectDetails.js`, `navigation.js`, и встроенный скрипт, который:
  1. Читает параметр `project` из URL.
  2. Находит проект в `projectDetails[slug]`.
  3. Собирает HTML и вставляет его в `#projectContent`.
  4. Если у проекта есть демо — вешает обработчик на форму и выводит результат в блок результата.

Ниже разобрано, как именно собирается HTML и как работает демо.

---

## Шаг 6. Как из URL достаётся проект

Во встроенном скрипте на `project.html`:

```javascript
var params = new URLSearchParams(window.location.search);
var slug = params.get('project');
```

- `window.location.search` — это строка параметров из адреса (например, `?project=healthylifestylega`).
- `URLSearchParams` разбирает эту строку.
- `params.get('project')` возвращает значение параметра `project`, то есть slug (например, `healthylifestylega`).

Дальше проверка:

```javascript
if (!slug || !window.projectDetails || !projectDetails[slug]) {
  contentEl.style.display = 'none';
  notFoundEl.style.display = 'block';
  return;
}
```

Если slug нет, или в `projectDetails` нет такого ключа — показывается «Проект не найден», контент скрывается. Иначе берётся объект проекта: `var project = projectDetails[slug];` — и с ним строится вся страница.

## Шаг 7. Сборка HTML по шагам (в том же скрипте)

Контент собирается в одну большую строку **html**, которая потом один раз записывается в `contentEl.innerHTML`. Порядок такой.

**7.1. Заголовок и подзаголовок**

```javascript
html += '<h1 class="project-detail__title">' + escapeHtml(project.title) + '</h1>';
if (project.subtitle) {
  html += '<p class="project-detail__lead">' + escapeHtml(project.subtitle) + '</p>';
}
```

- Заголовок страницы — всегда.
- Подзаголовок (lead) — если в данных есть `subtitle`.

**7.2. Технологии (теги)**

```javascript
if (project.techStack && project.techStack.length) {
  html += '<div class="project-detail__tech">';
  project.techStack.forEach(function(t) {
    html += '<span class="project-detail__tag">' + escapeHtml(t) + '</span>';
  });
  html += '</div>';
}
```

Для каждой технологии из `techStack` создаётся элемент с классом `project-detail__tag`. В CSS они оформлены как «таблетки».

**7.3. Кнопки действий**

```javascript
html += '<div class="project-detail__actions">';
html += '<a href="' + escapeHtml(project.githubUrl) + '" ...>View on GitHub</a>';
html += '<a href="index.html#projects" ...>Back to Projects</a>';
html += '</div>';
```

- Первая кнопка ведёт на GitHub.
- Вторая возвращает на главную в секцию проектов.

**7.4. Блок демо (форма «Попробовать»)**

Если `project.hasDemo` и есть `project.demoConfig`:

- Добавляется контейнер с `id="projectDemo"`.
- Заголовок и короткое описание демо.
- Форма с `id="demoForm"`:
  - Выпадающий список «Goal» — опции из `demoConfig.goals` (value и label).
  - Выпадающий список «Preference» — из `demoConfig.preferences`.
  - Кнопка «Get recommendation».
- Блок для результата с `id="demoResult"` (изначально пустой).

Все подписи и варианты берутся из данных, поэтому для другого проекта можно задать другие цели и предпочтения, просто изменив `demoConfig`.

**7.5. Текстовые секции**

Для каждой секции из `project.sections`:

```javascript
html += '<div class="project-detail__section">';
html += '<h3 class="project-detail__section-title">' + escapeHtml(s.title) + '</h3>';
html += '<p class="project-detail__section-p">' + escapeHtml(s.content) + '</p>';
if (s.bullets && s.bullets.length) {
  html += '<ul class="project-detail__bullets">';
  s.bullets.forEach(function(b) {
    html += '<li>' + escapeHtml(b) + '</li>';
  });
  html += '</ul>';
}
html += '</div>';
```

То есть: заголовок секции, абзац текста, затем при наличии — маркированный список.

**7.6. Как запустить и пример вывода**

Если в проекте есть `howToRun` и `exampleOutput`, добавляются ещё две секции: заголовок + блок с моноширинным текстом (класс `project-detail__code-block`).

**7.7. Вставка в страницу и обновление заголовка вкладки**

```javascript
contentEl.innerHTML = html;
contentEl.style.display = 'block';
document.title = project.title + ' | Arif Portfolio';
```

Так вся собранная разметка попадает в блок контента, и заголовок вкладки браузера отражает название проекта.

**Зачем `escapeHtml`?**  
Любой текст из данных (заголовки, абзацы, теги, варианты в форме) перед вставкой в HTML пропускается через функцию `escapeHtml`, чтобы специальные символы (`<`, `>`, `"` и т.д.) не превратились в разметку или скрипт. Это защита от XSS.

## Шаг 8. Как работает демо (форма и результат)

После того как HTML вставлен в страницу, если у проекта есть демо, скрипт находит форму и блок результата и вешает обработчик:

```javascript
if (project.hasDemo && project.demoConfig) {
  var form = document.getElementById('demoForm');
  var resultEl = document.getElementById('demoResult');
  var results = project.demoConfig.results;
  ...
}
```

**При отправке формы:**

1. **Отменяется стандартная отправка:**  
   `e.preventDefault();` — страница не перезагружается.

2. **Считываются выбранные значения:**  
   - цель: `document.getElementById('demoGoal').value` (например, `weight_loss`);  
   - предпочтение: `document.getElementById('demoPreference').value` (например, `vegetarian`).

3. **Формируется ключ для поиска результата:**  
   `var key = goal + '_' + pref;`  
   Например: `weight_loss_vegetarian`.

4. **Показывается состояние «Расчёт»:**  
   В блок результата записывается текст «Calculating...», блоку добавляется класс для видимости (например, плавное появление).

5. **Через 600 мс подставляется готовый результат:**  
   - Берётся объект `results[key]` из `demoConfig.results`.
   - Если он есть — в блок результата выводятся строки вида:  
     «Recommended diet: ...», «Recommended workout: ...», «Net calories: ...».
   - Если ключа нет — выводится сообщение вроде «No recommendation for this combination».

Python на сервере не вызывается: все варианты ответов заранее прописаны в `results`. Задержка в 600 мс нужна только для ощущения «расчёта».

## Шаг 9. Стили (CSS)

Все стили страницы проекта находятся в **`css/components.css`** в конце файла. Классы имеют префикс **`project-detail__`** и **`project-detail__demo`**.

**Основное:**
- **`.project-detail`** — отступы сверху и снизу (учёт фиксированной шапки).
- **`.project-detail .container`** — максимальная ширина контента (например, 800px), выравнивание по центру.
- **`.project-detail__back`** — ссылка «Назад к проектам».
- **`.project-detail__title`** — крупный заголовок (можно с градиентом).
- **`.project-detail__lead`** — подзаголовок: размер, цвет, ограничение ширины для удобства чтения.
- **`.project-detail__tech`** — контейнер для тегов (flex, перенос).
- **`.project-detail__tag`** — один тег технологии (фон, рамка, скругление).

**Секции и текст:**
- **`.project-detail__section`** — отступ между секциями.
- **`.project-detail__section-title`** — заголовок секции.
- **`.project-detail__section-p`** — абзац (цвет, межстрочный интервал).
- **`.project-detail__bullets`** — маркированный список (отступы, интервалы).
- **`.project-detail__code-block`** — блок для команды и примера вывода (моноширинный шрифт, фон, рамка).

**Демо:**
- **`.project-detail__demo`** — общий блок демо (карточка, стеклянный эффект).
- **`.project-detail__demo-title`** и **`.project-detail__demo-desc`** — заголовок и описание демо.
- **`.project-detail__demo-form`** — форма (flex, перенос, отступы).
- **`.project-detail__demo-row`** — одна строка: подпись + выпадающий список.
- **`.project-detail__label`** — подпись поля.
- **`.project-detail__select`** — выпадающий список (фон, рамка, фокус).
- **`.project-detail__demo-result`** — блок, куда выводится рекомендация; минимальная высота и плавное появление.
- **`.project-detail__demo-result--visible`** — класс для видимого состояния результата.
- **`.project-detail__demo-loading`** — стиль для текста «Calculating...».

Цвета, отступы и переменные (шрифты, радиусы и т.д.) берутся из общих стилей сайта (`variables.css` и др.), чтобы страница проекта выглядела в одном стиле с остальным портфолио.

## Шаг 10. Краткая последовательность при открытии страницы проекта

1. Пользователь на главной нажимает «View Project» на карточке Healthy Lifestyle GA.
2. Браузер переходит по адресу `project.html?project=healthylifestylega`.
3. Загружаются `projectDetails.js` и `navigation.js`, выполняется встроенный скрипт.
4. Скрипт читает `project=healthylifestylega` из URL.
5. В `projectDetails['healthylifestylega']` находится объект проекта.
6. Собирается HTML: заголовок, подзаголовок, теги, кнопки, блок демо (форма + результат), секции текста, «How to run», «Example output».
7. Собранный HTML записывается в `#projectContent`.
8. Для демо вешается обработчик на форму. При нажатии «Get recommendation» считываются цель и предпочтение, формируется ключ, через 0.6 с в блок результата подставляется соответствующий текст из `demoConfig.results`.

В итоге одна страница `project.html` и два источника данных (`projectsData` и `projectDetails`) обеспечивают работу страницы проекта и интерактивного демо. Чтобы добавить новый проект, достаточно добавить запись в `data.js` и объект с тем же slug в `projectDetails.js`, не создавая новых HTML-файлов.
